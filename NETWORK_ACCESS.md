# Мережевий доступ до React інтерфейсу

## Як дозволити іншим користувачам у мережі відкрити React компонент

### Налаштування виконано:

1. ✅ **Vite Dev Server** налаштований на прослуховування всіх мережевих інтерфейсів (`0.0.0.0:4201`)
2. ✅ **Backend API** налаштований на прослуховування всіх мережевих інтерфейсів (`0.0.0.0:4200`)
3. ✅ **CORS** налаштований для прийняття запитів з будь-якого джерела

### Як користуватися:

#### 1. Запустіть додаток на головному комп'ютері

Запустіть Desktop додаток (Insait_Translator_Deutsch.Desktop) у режимі Debug або Release.

#### 2. Дізнайтеся IP адресу головного комп'ютера

**Windows:**
```powershell
ipconfig
```
Шукайте рядок "IPv4 Address" - наприклад: `192.168.1.100`

**Linux/Mac:**
```bash
ip addr show
# або
ifconfig
```

#### 3. Підключіться з іншого пристрою

На іншому комп'ютері/телефоні/планшеті в тій же мережі відкрийте браузер та введіть:

```
http://[IP-АДРЕСА]:4201
```

Наприклад:
```
http://192.168.1.100:4201
```

### Можливі проблеми та рішення:

#### Брандмауер Windows блокує підключення

1. Відкрийте **Windows Defender Firewall** → **Advanced Settings**
2. Створіть нове правило для входящих підключень:
   - Port: 4200 та 4201
   - Protocol: TCP
   - Action: Allow
   - Profile: Domain, Private, Public

Або через командний рядок (PowerShell як адміністратор):
```powershell
New-NetFirewallRule -DisplayName "Insait Translator Backend" -Direction Inbound -LocalPort 4200 -Protocol TCP -Action Allow
New-NetFirewallRule -DisplayName "Insait Translator React UI" -Direction Inbound -LocalPort 4201 -Protocol TCP -Action Allow
```

#### Не вдається підключитися

1. Переконайтеся, що обидва пристрої в одній мережі
2. Перевірте, що брандмауер не блокує порти
3. Спробуйте пінгувати головний комп'ютер:
   ```
   ping 192.168.1.100
   ```
4. Перевірте, що додаток запущений і backend активний

#### Backend не відповідає

Перевірте консоль додатка на наявність повідомлень:
```
[NativeBackendHost] Started on http://0.0.0.0:4200/
```

### Режим розробки (Dev Server)

У режимі Debug автоматично запускається React Dev Server на порту 4201.

Користувачі з мережі можуть відкрити:
- **React UI (Hot Reload)**: `http://[IP]:4201`
- **Backend API**: `http://[IP]:4200/api/health`

### Production режим

У Release режимі React UI вбудовано в додаток і обслуговується backend'ом на порту 4200:
- **UI + API**: `http://[IP]:4200`

### Налаштування змінних оточення (опціонально)

Можна змінити порти через змінні оточення:

```powershell
$env:INSAIT_UI_URL = "http://0.0.0.0:5000/"
$env:INSAIT_UI_DEV_URL = "http://0.0.0.0:5001/"
```

### Безпека

⚠️ **Важливо**: 
- Це налаштування призначене для локальної мережі
- Не відкривайте порти в Інтернет без додаткової автентифікації
- У публічних мережах використовуйте VPN або інші засоби захисту

### Тестування

1. На головному ПК відкрийте: `http://localhost:4201`
2. На іншому пристрої відкрийте: `http://[IP-ГОЛОВНОГО-ПК]:4201`
3. Спробуйте перекласти текст - він має працювати з обох адрес

---

**Примітка**: Для постійного використання рекомендується налаштувати статичну IP адресу на головному комп'ютері в налаштуваннях роутера.
