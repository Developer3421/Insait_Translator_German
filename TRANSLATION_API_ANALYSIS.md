# Аналіз API перекладу - Insait Translator Deutsch

## Огляд використовуваних сервісів

У додатку використовуються **3 різні провайдери** для перекладу тексту:

| Провайдер | Тип | API Endpoint | Ключ API |
|-----------|-----|--------------|----------|
| MyMemory | Безкоштовний REST API | `api.mymemory.translated.net` | Не потрібен |
| GTranslate | Unofficial Google Translate | Різні endpoints Google | Не потрібен |
| Google Gemini | Офіційний API | `generativelanguage.googleapis.com` | **Потрібен** |

---

## 1. MyMemory API

### Характеристики
- **Endpoint**: `https://api.mymemory.translated.net/get`
- **Метод**: GET запит
- **Авторизація**: Не потрібна (анонімний доступ)
- **Ліміти**: ~5000 символів/день для анонімних користувачів

### Чи можуть виявити використання з кінцевих пристроїв?

**ТАК, можуть виявити:**

1. **IP-адреса**: Запити йдуть безпосередньо з IP-адреси користувача
2. **Відсутність User-Agent**: У коді не встановлено User-Agent заголовок
3. **Географічне розташування**: IP-адреси з різних країн/регіонів
4. **Патерни запитів**: Різні IP роблять схожі запити

### Що бачить MyMemory:
```
IP: [IP користувача]
User-Agent: (не встановлено - буде стандартний .NET HttpClient)
Час: [час запиту]
Текст: [текст для перекладу]
```

### Ризики:
- ⚠️ **Низький ризик** - MyMemory є публічним безкоштовним API
- ✅ Немає порушення ToS при використанні з десктоп додатку
- ⚠️ Але: кожен користувач має свій ліміт в 5000 символів/день

## 2. GTranslate (Unofficial Google Translate)

### Характеристики
- **Бібліотека**: `GTranslate` NuGet пакет v2.3.1
- **Тип**: Реверс-інженерія Google Translate web interface
- **Авторизація**: Не потрібна

### Чи можуть виявити використання з кінцевих пристроїв?

**ТАК, ВИСОКИЙ РИЗИК ВИЯВЛЕННЯ:**

1. **IP-адреса**: Запити йдуть з IP користувача до серверів Google
2. **Fingerprinting**: Google має передові методи виявлення автоматизації
3. **Rate limiting**: Google може блокувати IP при підозрілій активності
4. **Патерни запитів**: Автоматизовані запити легко відрізнити від браузерних

### Як працює GTranslate:
- Емулює запити до `translate.google.com`
- Google може виявити що це не браузер
- Може з'явитись CAPTCHA або блокування IP

### Ризики:
- 🔴 **ВИСОКИЙ РИЗИК** - Це неофіційний клієнт
- 🔴 **Порушення ToS** - Google Terms of Service забороняють автоматизований доступ без API
- 🔴 **Блокування**: Користувачі можуть отримати бан по IP
- 🔴 **Нестабільність**: Може перестати працювати в будь-який момент

### Що Google може бачити:
```
IP: [IP користувача]
User-Agent: (GTranslate встановлює свій UA)
Cookies: Можливо відсутні або неправильні
TLS Fingerprint: Не браузерний
Частота: Підозрілі патерни
```

## 3. Google Gemini API (Офіційний)

### Характеристики
- **Endpoint**: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent`
- **Модель**: `gemini-2.5-flash`
- **Авторизація**: API Key в URL параметрі

### Чи можуть виявити використання з кінцевих пристроїв?

**ТАК, АЛЕ ЦЕ НЕ ПРОБЛЕМА:**

1. **IP-адреса**: Google бачить IP користувача
2. **API Key**: Ваш ключ ідентифікує проект
3. **Billing**: Витрати прив'язані до вашого акаунту

### Що Google бачить:
```
API Key: [ваш ключ]
IP: [IP користувача]
Project: [ваш Google Cloud проект]
Usage: Кількість токенів, запитів
```

### Ризики:
- ✅ **Легальне використання** - Це офіційний API
- ⚠️ **Витрати**: Якщо ключ вбудований в додаток - всі витрати на вас
- 🔴 **Зловживання**: Хтось може витягти ключ і перевитратити квоту
- ⚠️ **Rate limits**: Можуть бути перевищені при багатьох користувачах

### Безпека ключа:
У поточній реалізації API ключ:
- Зберігається в `SettingsService`
- Передається в URL як query параметр (`?key=...`)
- ⚠️ Може бути перехоплений (хоча HTTPS шифрує URL)

## Рекомендації

### 1. Для MyMemory
- ✅ Можна залишити як є
- 💡 Додати User-Agent заголовок для кращої ідентифікації

### 2. Для GTranslate (КРИТИЧНО)
- 🔴 **Це найбільший ризик** - Google може заблокувати IP користувачів
- 💡 Розглянути заміну на легальну альтернативу:
  - LibreTranslate (self-hosted)
  - DeepL API (платний, але з безкоштовним тарифом)
  - Microsoft Translator API

### 3. Для Google Gemini
- ⚠️ Якщо API ключ вбудований в додаток:
  - Хтось може його витягти
  - Всі витрати падуть на вас
- 💡 Краще: вимагати від користувачів власні ключі
- 💡 Або: проксі сервер з вашим ключем + rate limiting

## Технічні деталі виявлення

### Що бачать сервіси при запиті:

```
HTTP Request Headers:
- IP-адреса: IP кінцевого користувача (не ваш сервер)
- User-Agent: (залежить від бібліотеки)
- Accept-Language: (може виявити локаль системи)

TLS Fingerprint:
- .NET HttpClient має відмінний TLS fingerprint від браузерів
- Google може виявити що це не браузер

Патерни використання:
- Багато різних IP з однаковими патернами
- Відсутність cookies/сесій
- Відсутність JavaScript виконання
```

## Висновок

**Усі три сервіси бачать IP-адреси кінцевих користувачів, не вашого серверу.**

Найбільша проблема - **GTranslate**, яка порушує Terms of Service Google і може призвести до блокування IP-адрес ваших користувачів.

MyMemory і Gemini API - легальні рішення, але мають обмеження по використанню.
